# Contributing

First off, thank you for considering contributing. As a component of the AXIONYX ecosystem, we hold contributions to strict scientific and engineering standards.

## The AXIONYX Engineering Philosophy

Before contributing, please understand our core principles:
1. **Evidence before conclusions**: Code must be backed by tests and theoretical validation.
2. **Provenance preservation**: Commits should clearly state *why* a change was made and link to relevant domain knowledge.
3. **Traceability**: Ensure that new capabilities can be audited and traced back to requirements.

## How to Contribute

### 1. Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the expected behavior, the actual behavior, and steps to reproduce.

### 2. Suggesting Enhancements
- Open a discussion or an enhancement issue.
- Clearly articulate how the enhancement aligns with the core methodology.

### 3. Pull Requests
- Fork the repository and create your branch from `main`.
- Ensure your code adheres to our architectural guidelines.
- Include unit tests and validation evidence.
